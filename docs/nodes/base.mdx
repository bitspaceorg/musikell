There are four categories of base nodes in Musikell:

1. Creational
2. Conditional
3. Manipulative
4. Topological

## Creational

Creational nodes are responsible for generating sound from scratch (e.g., oscillators, kicks, hi-hats, and more).

### Input

| Field     | Type   | Required | Description                                |
| --------- | ------ | -------- | ------------------------------------------ |
| Node Type | string | Yes      | Specifies the type of node to create.      |
| Amplitude | number | No       | Defaults to `1`. Controls output loudness. |

### Output

| Field  | Type           | Description                                                          |
| ------ | -------------- | -------------------------------------------------------------------- |
| signal | `byte[256][N]` | An array of 256-byte signal buffers for efficient conduit streaming. |
| meta   | object         | Metadata produced by the node type.                                  |
| parent | object \| null | Reference to the parent node, if one exists.                         |

## Conditional

Conditional nodes are responsible for evaluating a threshold value based on their condition type.  
They output either `[0.0..0.0]` or the input signal(s), depending on the result (e.g., if-else, switch, and more).

### Input

| Field     | Type           | Description                                                          |
| --------- | -------------- | -------------------------------------------------------------------- |
| signal    | `byte[256][N]` | An array of 256-byte signal buffers for efficient conduit streaming. |
| Node Type | string         | Specifies the type of conditional node.                              |
| Amplitude | number         | Optional. Defaults to `1`. Controls output amplitude.                |

### Output

| Field  | Type           | Description                                                            |
| ------ | -------------- | ---------------------------------------------------------------------- |
| signal | `byte[256][N]` | An array of 256-byte signal buffers representing the processed output. |
| meta   | object         | Metadata produced by the node.                                         |
| parent | object \| null | Reference to the parent node, if one exists.                           |

## Manipulative

Manipulative nodes modify the incoming signal according to their specific processing type (e.g., reverb, delay, and more).

### Input

| Field     | Type           | Description                                                          |
| --------- | -------------- | -------------------------------------------------------------------- |
| signal    | `byte[256][N]` | An array of 256-byte signal buffers for efficient conduit streaming. |
| Node Type | string         | Specifies the type of manipulative node.                             |
| Amplitude | number         | Optional. Defaults to `1`. Controls output amplitude.                |

### Output

| Field  | Type           | Description                                                            |
| ------ | -------------- | ---------------------------------------------------------------------- |
| signal | `byte[256][N]` | An array of 256-byte signal buffers representing the processed output. |
| meta   | object         | Metadata produced by the node.                                         |
| parent | object \| null | Reference to the parent node, if one exists.                           |

## Topological

Topological nodes allow you to encapsulate a collection of nodes into a single reusable unit.  
They act like _subgraphs_, _groups_, or _modules_ that can be dropped anywhere in the workflow.  
A topology can:

- group multiple nodes into one logical unit
- expose clean entry and exit points
- hide internal complexity
- support nesting (topology inside topology)
- behave like an “importable module” for larger workflows
- allow parameter overrides when used in different contexts
- standardize reusable chains (e.g., “default drum chain”, “mastering chain”)

### Input

### Input

| Field     | Type   | Description                                                                                                            |
| --------- | ------ | ---------------------------------------------------------------------------------------------------------------------- |
| entry     | node   | The actual first node of the topology. Whatever signal is fed into this node becomes the input of the entire topology. |
| exit      | node   | The final node of the topology. Its output becomes the topology’s exposed output.                                      |
| params    | object | Optional external parameters passed into the topology (e.g., wetness, decay, gain).                                    |
| Amplitude | number | Optional. Defaults to `1`. Final amplitude applied after the exit node.                                                |

### Output

| Field  | Type           | Description                                                                    |
| ------ | -------------- | ------------------------------------------------------------------------------ |
| signal | `byte[256][N]` | The final processed signal produced by the topology’s `exit` node.             |
| meta   | object         | Metadata aggregated from the topology, primarily derived from the `exit` node. |
| parent | object \| null | Reference to the node that instantiated this topology, if any.                 |
